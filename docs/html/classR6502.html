uint8_t R6502::ABS ( )
private

Absolute addressing mode.

Second byte has low order byte of effective address, third has higher order byte (reminder again: 6502 is little endian!!!) That means ABS instructions can acess all 64KB of address space. 2^8 = 0x100 2 byte addressing... 0x100 * 0x100 = 0x10000 (64KB)

Returns
uint8_t Extra cycle possible?

uint8_t R6502::IND ( )
private

Indirect addressing mode.

Things start to look weird here in order to simulate a bug in the CPU. From hnesdev.icequake.net/6502bugs.txt: "An indirect JMP (xxFF) will fail because the MSB will be fetched from address xx00 instead of page xx+1." DO NOT simplify this. Need to make the bug clear and show details.
